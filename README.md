# 经营问题诊断与 SOP 助手

面向 AI 产品/运营岗位的前端作品集项目。它不是聊天框，而是一条可以演示和验收的结构化业务闭环：

`工单录入 → 智能诊断 → 人工确认 → SOP 生成/编辑 → 评测 → 看板复盘`

## 功能

- 30 条脱敏异常工单，覆盖客户投诉、订单履约、系统异常、项目交付和内部流程。
- 问题分类、两级原因树、责任角色和可追溯引用来源。
- 四项诊断结果必须逐项人工确认，确认完成后才能生成 SOP。
- SOP 支持编辑、版本保存、发布校验和浏览器本地持久化。
- 20 条分类一致性评测与 10 条 SOP 完整性抽查。
- 默认使用稳定的模拟 AI，并提供 OpenAI 兼容接口适配器。

## 本地运行

```bash
pnpm install
pnpm dev
```

生产构建与测试：

```bash
pnpm test
pnpm build
```

## 演示建议

1. 从概览看板说明五类经营异常和当前处理进展。
2. 新建一条工单，在异常详情中启动智能诊断。
3. 修改 AI 建议的最终值，逐项完成人工确认。
4. 生成 SOP，说明责任人、升级条件、结束条件和引用来源。
5. 保存新版本并发布，然后回到评测页解释验收方法。

## 数据与安全

- 编辑结果保存在当前浏览器的 `localStorage` 中。
- “恢复演示数据”会覆盖浏览器中的现有演示状态，并在操作前二次确认。
- API 密钥不写入代码或长期存储。`OpenAICompatibleProvider` 只接受运行时配置；纯前端直连还需目标接口允许浏览器跨域请求。
- 公开知识来源仅保存短摘要和链接，不复制大段原文。
